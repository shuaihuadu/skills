# Video Creator 视频创作技能

创建专业的短视频，包含动画场景、TTS 语音旁白和字幕。每个视频的视觉风格由 AI 根据主题量身定制。

## 概述

此技能通过以下步骤生成 1-3 分钟的 MP4 视频：

1. AI 生成 `video-config.json`（内容、旁白、时间线）
2. AI 生成 `src/index.html`（HTML/CSS/JS 动画，视觉风格匹配主题）
3. 运行 `generate_tts.py`，通过 edge-tts 生成音频和 SRT 字幕
4. 运行 `build_video.py`，在无头浏览器中录制动画，通过 FFmpeg 合并音视频

**关键理念**：AI 同时掌控内容和视觉两层，每个视频都有独特的设计。

## 用户只需提供

| 参数     | 说明                                    | 默认值   |
| -------- | --------------------------------------- | -------- |
| **主题** | 视频讲什么                              | （必填） |
| **语言** | `中文` 或 `英文`                        | 中文     |
| **声音** | `男声` 或 `女声`                        | 男声     |
| **时长** | `短`(1分钟) / `中`(2分钟) / `长`(3分钟) | 中       |

**示例对话**：

> 用户：帮我做一个介绍 Rust 语言的视频，英文女声，短一点

> 用户：做一个关于光合作用原理的科普视频

> 用户：帮我介绍一下文艺复兴时期的绘画艺术，中文女声

> 用户：Make a video about Japanese tea ceremony, English

就这些！场景编排、旁白撰写、视觉设计全部由 AI 自动完成。

## 工作流程

### 第 1 步：收集需求

只问用户**主题**。其他参数有默认值，用户不提就不问。

### 第 2 步：AI 自动规划内容

根据主题自动完成以下工作（**无需询问用户**）：

1. **调研主题**：了解核心要点、关键事实、典型素材（概念、人物、事件、数据等）
2. **编排场景**：确定场景数量和每个场景要展示的内容，构建叙事节奏
3. **撰写旁白**：用自然口语化的语气写旁白（像在讲故事，不是念稿子）
4. **计算时长**：按旁白字数估算每个场景的 duration
5. **确定视觉风格**：根据主题的气质和情感选择配色、动画风格、布局方案

### 第 3 步：生成 video-config.json

在项目工作目录中创建 `video-config.json`。

**重要**：将文件放在项目工作目录中（与 `src/` 和 `output/` 同级）。

#### video-config.json 最小必须字段

构建脚本只依赖以下字段：

```json
{
  "title": "视频标题",
  "voice": "zh-CN-YunxiNeural",
  "scenes": [{ "narration": "旁白文字", "duration": 10000 }]
}
```

| 字段                 | 类型   | 必须 | 说明                   |
| -------------------- | ------ | ---- | ---------------------- |
| `title`              | string | 是   | 视频标题               |
| `voice`              | string | 是   | edge-tts 声音名称      |
| `scenes`             | array  | 是   | 场景数组               |
| `scenes[].narration` | string | 是   | 旁白文本（用于 TTS）   |
| `scenes[].duration`  | number | 是   | 场景时长（毫秒）       |
| `language`           | string | 否   | 语言代码（默认 zh-CN） |
| `voiceRate`          | string | 否   | 语速（默认 -5%）       |
| `voiceVolume`        | string | 否   | 音量（默认 +0%）       |

**自定义字段**：AI 可以在 config 中添加任意字段（如 `heading`、`type`、`colors`、
`items`、`codeSnippet` 等），供自己生成的 HTML 读取来驱动渲染。
构建脚本完全不关心这些字段。

### 第 4 步：生成 HTML 动画

这是最关键的一步。AI 在项目的 `src/index.html` 中生成**完整的单文件 HTML**
（内联 CSS 和 JS），包含所有场景的动画效果。

> **重要**：每个视频的视觉设计必须是独特的。不要套用固定模板。
> 根据主题的气质选择完全不同的配色、布局、动画和过渡效果。

#### 必须遵守的 4 个约定

只要满足这 4 个约定，其余设计完全自由。
完整合约文档见 [references/HTML-CONTRACT.md](references/HTML-CONTRACT.md)。

1. **headless 自动播放**：检测 `?headless=true` 参数时立即播放，不显示按钮
2. **播放信号**：播放开始瞬间设置 `window.__playbackStartedMs = performance.now()`
3. **字幕容器**：页面有 `<div id="subtitle-bar"><span id="subtitle-text"></span></div>`，
   从 `../assets/audio/scene_N.srt` 加载字幕，headless 下用全局 wall-clock 时间驱动
4. **音频路径**：`../assets/audio/scene_N.mp3` 和 `.srt`（HTML 在 `src/` 下）

分辨率固定 **1920×1080**。场景时长从 `../video-config.json` 的 `scenes[].duration` 读取。

#### 创意设计要求

**不要照搬任何模板或示例代码的视觉风格。** 每次生成 HTML 时，必须根据主题从零设计：

- **配色方案**：从主题的气质和情感推导，而非默认暗色科技蓝。
  技术主题用品牌色，自然科学用清新色调，历史人文用暖色复古，
  商业用专业深蓝金，美食用暖色系，艺术用高饱和对比色
- **布局方式**：灵活运用 flex、grid、绝对定位、多列布局、杂志排版。
  不同场景可以有不同布局（不必每个场景都居中堆叠）
- **动画技术**：CSS `@keyframes`、`transition`、Canvas 2D/WebGL、SVG 动画、
  requestAnimationFrame 驱动的自定义动画。选择匹配主题气质的技术
- **背景效果**：渐变色、粒子系统、几何网格、波浪、星空、水墨晕染、
  纸张纹理、摄影虚化、光影变化等 — 根据主题选择
- **场景过渡**：淡入淡出、滑动、缩放、翻转、形变、百叶窗、水波纹等。
  不要每个场景都用相同过渡
- **字体**：可用 Google Fonts 加载特色字体。技术主题用等宽字体，
  人文主题用衬线字体，现代主题用无衬线字体，艺术主题用手写字体
- **独特元素**：根据主题选择 — 代码打字、终端模拟、数据图表、
  时间线、地图路径、画笔动画、分子模型、人物肖像、
  手绘插图、波形可视化、3D 透视变换等

#### 结构多样性要求

**不要每个视频都用"N 个全屏 section + display:none/block 切换 + 居中 flex 布局"的固定结构。**
场景的 DOM 组织方式和切换机制本身应该因主题而异：

**场景组织方式**（每次选择不同的方案）：

- **全屏层叠切换**：场景用 `position:absolute` 层叠，通过 opacity / transform 过渡
- **横向/纵向滚动**：所有场景排列在一条轴上，用 `transform:translateX/Y` 切换
- **单容器内容替换**：同一个容器，每次场景切换时 JS 动态替换内容 + 入场动画
- **Canvas 全程绘制**：整个视频用一个 Canvas，场景切换 = 重绘画面
- **多区域并行**：屏幕分成多个区域，不同区域独立更新（如左侧主内容 + 右侧数据面板）
- **时间线驱动**：所有元素共存于同一 DOM，通过 CSS animation-delay 按时间线出现/消失
- **3D 空间切换**：场景作为 3D 平面，用 CSS `perspective` + `rotateY/X` 翻转切换

**布局架构**（不要每个场景都是"标题+内容居中"）：

- 杂志排版（图文混排、多列、不对称）
- 仪表盘布局（多 panel、网格、数据卡片阵列）
- 全出血大图 + 覆盖文字
- 分屏对比（左右/上下对比展示）
- 卡片/瀑布流（多个卡片交错出现）
- 时间线/路径（内容沿线性路径排列）
- 终端/代码编辑器（模拟具体应用界面）

#### 动画生动性要求

静态页面切换是枯燥的。**每个场景内部必须有持续的动态效果**，让画面始终"活"着：

**元素入场动画**（每个场景切入时）：

- 标题/文字：打字机逐字显现、从底部滑入、模糊渐清、弹性缩放弹入
- 列表/卡片：依次交错入场（stagger），每项间隔 200-400ms
- 图表/数据：数值从 0 增长到目标值、柱状图逐根升起、饼图扇形展开
- 图片/图标：淡入+微缩放、从侧面飞入、翻转显现
- 代码块：逐行打字、语法高亮逐色点亮

**持续运动效果**（场景展示期间保持动态）：

- **背景层**：粒子缓慢漂移、渐变色缓慢旋转、波纹持续扩散、
  星空缓慢闪烁、几何图形缓慢旋转/变形
- **装饰元素**：浮动的图标/符号缓慢上下摆动、光晕呼吸闪烁、
  连线上的流动光点、环绕轨道运动的小元素
- **内容区微动**：卡片轻微悬浮感（微妙的上下浮动）、
  重点数字的脉冲高亮、进度条/时间轴的持续推进

**场景间过渡**（不要硬切）：

- 每次场景切换都要有过渡动画（300-800ms）
- 相邻场景用不同过渡：滑动、淡入淡出、缩放、翻页、溶解等交替使用
- 可以用 CSS `transition`/`animation` 或 JS 时间控制实现

**动画节奏原则**：

- 入场动画快速有力（200-600ms），抓住注意力
- 持续效果缓慢柔和（2-10s 循环），不分散注意力
- 关键数据/要点出现时用强调动画（脉冲、闪光、放大）
- 使用 `cubic-bezier` 或 `ease-out` 等缓动函数，避免线性运动的机械感

### 第 5 步：搭建环境

在项目工作目录中运行以下命令：

```bash
python -m venv .venv
# Windows:
.venv\Scripts\activate
# macOS/Linux:
# source .venv/bin/activate

pip install edge-tts playwright
playwright install chromium
```

同时确保 FFmpeg 已安装并在 PATH 中可用。

### 第 6 步：生成 TTS 音频

```bash
python <skill-path>/scripts/generate_tts.py video-config.json
```

此命令读取配置，为每个场景生成 MP3 和 SRT 字幕，并写入 `assets/audio/` 目录。

### 第 7 步：构建视频

```bash
python <skill-path>/scripts/build_video.py video-config.json
```

此命令会：

1. 确认 `src/index.html` 存在
2. 启动本地 HTTP 服务器
3. 在无头 Chromium 中录制动画（等待 `__playbackStartedMs` 信号）
4. 裁剪页面启动延迟
5. 拼接并填充音频
6. 合成最终 MP4

最终 MP4 文件位于 `output/<timestamp>/video.mp4`。

### 第 8 步：交付给用户

告知用户输出文件的位置和大小。

## 时长计算规则

中文 TTS 约每秒 4 个字；英文约每秒 2.5 个单词。每个场景额外留 2-3 秒动画缓冲。

## 使用技巧

- 视频总时长建议控制在 3 分钟以内，效果最佳
- 使用 5-8 个场景，节奏较为均衡
- 旁白语气要自然、口语化，避免机械感
- 每个场景只传达一个核心要点
- 根据主题选择配色和动画风格，让每个视频都有辨识度
- 分辨率固定 1920×1080，所有布局按此设计
- HTML 建议单文件（内联 CSS/JS），减少加载依赖
- 场景内必须有持续动态效果（背景运动、元素浮动），避免静态幻灯片感
- 元素入场用交错动画（stagger），比同时出现更有层次感
- 善用 CSS `cubic-bezier` 缓动曲线，让运动有弹性和节奏感
- HTML 建议单文件（内联 CSS/JS），减少加载依赖
